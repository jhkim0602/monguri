## 배경
현재 멘토 영역은 UI 중심의 mock 데이터로 구성되어 있으며, 멘티 영역(과제/플래너/캘린더/칼럼)과 실제 데이터 연동이 없다. 멘토가 해야 할 핵심 업무(과제 생성/배정, 플래너/캘린더 확인, 코멘트/피드백, 칼럼 작성)를 단일 워크플로로 제공하도록 전면 리팩토링이 필요하다.

## 결정 사항
- 멘토는 랜딩 페이지에서 로그인 후 `/mentor`로 진입한다.
- 기존 멘토 페이지(`src/app/(mentor)/mentor/*`)는 전면 삭제 후 재구성한다.
- 초기 구현은 **멘티 mock 데이터**를 재사용하고, 이후 API 연동으로 쉽게 교체되도록 분리한다.

## 목표(Goals)
1. 멘토가 **과제 생성/배정/피드백**을 한 곳에서 처리할 수 있다.
2. 멘토가 **학생 캘린더**와 **플래너 모아보기(주간/월간)** 를 조회하고 코멘트할 수 있다.
3. 멘토가 **설스터디 칼럼(서울대쌤 칼럼)** 을 작성/발행하고 멘티 칼럼 페이지에 반영된다.
4. 멘티 영역의 기존 UI를 유지하면서 **데이터 소스를 API 기반**으로 전환할 수 있는 기반을 만든다.
5. 기존 mentor mock 페이지를 제거/교체하더라도 **기능 단위로 재사용 가능한 컴포넌트 구조**를 제공한다.

## 비목표(Non-Goals)
- 결제/정산, 권한 과금 모델
- 라이브 강의/채팅의 실시간 기능(1차 범위에서 제외)
- 고급 추천/AI 기반 피드백 자동화

## 사용자/페르소나
- 멘토(Primary): 과제 설계/배정, 학습 기록 확인, 피드백 작성, 칼럼 작성
- 멘티(Secondary): 멘토가 만든 과제/피드백/칼럼을 수신
- 운영자(Admin): 멘토/멘티 관리, 콘텐츠 관리(후속 범위)

## 핵심 사용자 시나리오
1. **과제 생성/배정**
   - 멘토가 과제 템플릿을 작성 → 학생/그룹에 배정 → 마감일 설정 → 멘티 플래너/캘린더에 표시
2. **플래너/캘린더 리뷰**
   - 멘토가 학생 캘린더/플래너 모아보기에서 완료 기록을 확인 → 코멘트 작성 → 멘티 홈/플래너에 반영
3. **피드백 제공**
   - 멘토가 과제 제출물/질문을 확인 → 피드백/보완 요청 → 상태 변경(approved/revise)
4. **칼럼 작성/발행**
   - 멘토가 칼럼 초안을 작성 → 시리즈/태그 설정 → 공개 발행 → 멘티 칼럼 페이지에 노출

## 기능 요구사항(Functional Requirements)
### A. 멘토 대시보드
- 오늘/주간 지표: 대기 중 피드백, 제출물 수, 미팅 예정, 최근 활동 학생
- 중요 알림: 미제출 과제, 코멘트 요청, 승인 대기

### B. 학생 관리
- 학생 리스트: 검색/필터(학년, 과목, 상태)
- 학생 상세: 프로필/목표/성취도, 과제 이력, 플래너/캘린더 탭

### C. 과제(Task) 관리
- 과제 생성(템플릿): 제목, 설명, 과목, 첨부, 난이도, 예상 시간
- 학생 배정: 개별/그룹/전체, 마감일, 시작/종료 시간, 반복 설정
- 상태 흐름: pending → submitted → feedback_completed

### D. 플래너/캘린더 리뷰
- 학생 캘린더 월간/주간 뷰
- 플래너 모아보기(주간/월간 카드) 조회
- 플래너 코멘트/인증 피드백 작성

### E. 피드백/코멘트
- 제출물 상세 보기(텍스트/이미지/파일)
- 승인/보완 요청 플로우
- 코멘트 히스토리 및 상태 변화 로그

### F. 설스터디 칼럼(서울대쌤 칼럼)
- 시리즈 관리(생성/수정/비공개)
- 칼럼 작성(초안/발행/예약)
- 멘티 칼럼 페이지에 노출/검색/북마크 연동

### G. 글작성(공지/게시글)
- 멘토 공지/게시글 작성 및 학생에게 노출(선택 범위)

## 데이터 요구사항
- 멘티 플래너/캘린더는 멘토 과제/피드백을 실시간 반영해야 한다.
- 기존 localStorage 키(`planner-day-tasks`, `mentee-calendar-events`, `mentor-meeting-records`)는 API로 이관 계획을 갖는다.

## UX/품질 요구사항
- 멘토 동선 단축(리스트 → 상세 → 피드백 입력 최소 클릭)
- 모바일/태블릿 대응
- 로딩/에러/빈 상태 정의

## 성공 지표
- 멘토 1명 기준: 과제 생성 → 배정 완료까지 2분 이내
- 피드백 처리 평균 시간 30% 감소
- 멘티 칼럼 조회수 증가

## 수용 기준(acceptance)
- 멘토가 과제 생성/배정/피드백을 수행하면 멘티 플래너/캘린더에 반영된다.
- 멘토가 플래너 모아보기에서 코멘트를 남길 수 있다.
- 칼럼 작성/발행이 멘티 칼럼 리스트에 반영된다.
- 기존 mentor mock 페이지/컴포넌트는 제거 또는 대체된다.

## 제약/가정
- 프론트는 Next.js App Router + Tailwind 기반 유지
- 백엔드는 추후 확정(REST 기반 계약 우선 정의)
